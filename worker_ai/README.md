# Worker Service

Автономный сервис обработки отзывов. Папка `/worker` содержит все необходимое для отдельного запуска без доступа к БД приложения.

## Что делает сервис

- Каждые `WORKER_POLL_INTERVAL` секунд опрашивает сайт
- Получает отзывы через HTTP API приложения
- Находит новые отзывы со статусом `new`
- Сразу определяет тональность отзыва
- Отправляет уведомление о новом отзыве в Telegram
- Генерирует ответ через OpenAI API
- При ошибке OpenAI или отсутствии ключа использует локальную fallback-логику
- Создает в приложении отдельный комментарий-ответ от ИИ на каждый новый отзыв или комментарий
- Помечает исходный отзыв и созданный ИИ-комментарий как обработанные
- Не отвечает на собственные сообщения, чтобы не запускать бесконечную цепочку автоответов

## Файлы

- `config.py` — настройки через `.env`
- `client.py` — HTTP-клиент для работы с приложением
- `models.py` — модели данных worker
- `processor.py` — OpenAI и fallback-логика
- `state.py` — локальное состояние обработанных уведомлений и отзывов
- `telegram_bot.py` — отправка уведомлений в Telegram
- `worker.py` — основной цикл
- `requirements.txt` — зависимости
- `Dockerfile` — сборка контейнера
- `docker-compose.yml` — запуск отдельного контейнера

## Настройки

Пример `.env`:

```env
TARGET_SITE_URL=http://127.0.0.1:8000
WORKER_API_TOKEN=change-me
WORKER_POLL_INTERVAL=10
STATE_FILE_PATH=data/state.json
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4.1-mini
AI_AUTHOR_NAME=AI Support
TELEGRAM_BOT_TOKEN=
TELEGRAM_USER_CHAT_ID=
```

Если вы работаете без DNS, укажите IP-адрес прямо в `TARGET_SITE_URL`, например `http://192.168.1.20:8000`.

## Запуск

Из директории `/worker`:

```bash
docker compose up --build
```

Важно: `WORKER_API_TOKEN` должен совпадать со значением `WORKER_API_TOKEN` в `/app/.env`, иначе приложение не позволит worker обновлять отзывы.

Локально:

```bash
pip install -r requirements.txt
python worker.py
```

## Telegram

Чтобы получать уведомления о новых отзывах:

1. Создайте бота через `@BotFather`
2. Получите `TELEGRAM_BOT_TOKEN`
3. Откройте личный чат с ботом и нажмите `/start`, чтобы бот смог писать пользователю
4. Узнайте `TELEGRAM_USER_CHAT_ID` этого личного чата
5. Укажите оба значения в `.env`

Для обратной совместимости все еще поддерживается старый параметр `TELEGRAM_CHAT_ID`, но для личных уведомлений лучше использовать `TELEGRAM_USER_CHAT_ID`.

Если Telegram не настроен, `worker` продолжит работу без отправки уведомлений.
